# Plan: Update Godot API Download to Use Correct Tags

## Goal
Modify the `downloadApi` and `generate` tasks in `build.mill` to download Godot extension API files using the appropriate Git tag instead of assuming a stable branch, ensuring version-specific downloads.

## Background
- Current implementation uses `[major.minor]-stable` as branch, e.g., `4.5-stable` for version `4.5.0`.
- From Godot's GitHub tags, the initial release uses `major.minor-stable` (e.g., `4.5-stable` for `4.5.0`), but patch releases use `major.minor.patch-stable` (e.g., `4.5.1-stable`).
- For main/development, use `main` branch.

## Steps
1. Update the branch/tag calculation logic in both `downloadApi` and `generate` tasks.
   - For version "main", use "main".
   - For versions like "4.5.0", if patch is "0", use "4.5-stable"; else use "4.5.1-stable".
2. Test the updated logic by running downloadApi for a few versions (e.g., 4.5.0, 4.6.1, main).
3. Verify the downloaded files are correct and usable by the generator.

## Risks
- If a version doesn't have a matching tag, the download will fail (curl -f will exit with error).
- Ensure URLs are accessible; if not, may need alternative sources.

## Validation
- Run `mill 'gdext.`generator-module`.downloadApi' 4.5.0` and check if files download to `gdext/generator-module/resources/4.5.0/`.
- Similarly for 4.6.1 and main.
