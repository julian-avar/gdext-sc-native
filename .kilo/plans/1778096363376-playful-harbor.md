# Plan: Diagnose and Fix Printing Issue in Scala Godot Example

## Diagnosis
The code in `example/example_scene_scala.scala` is correct. The extension loads successfully, the class is registered, and instances are created. However, the `_ready()` method is not being called because no Godot scene is loaded that uses the `ExampleSceneScala` class.

- `GdxApi.printString` works (as evidenced by successful binding loading)
- Scala `println` works (as seen in the logs from `GodotEntry`)
- The issue is that `_ready()` is only called when the node is part of an active scene tree

## Root Cause
In Godot, `_ready()` is called when a node is added to the scene tree and becomes ready. Since the user is running Godot without loading a specific scene that uses `ExampleSceneScala`, no instances are added to the scene tree, so `_ready()` is never invoked.

## Fix
To test the example and see "Hello, Scala!" printed:

1. Open Godot Editor
2. Create a new scene (Scene > New Scene)
3. Add a CenterContainer node as the root (Node > UI > CenterContainer)
4. Select the CenterContainer node
5. In the Inspector, under "Script", click "New Script"
6. Select "C#" but change to "Other" and browse to the compiled extension (though since it's Scala, attach via code or find the generated class)
7. Actually, since it's a GDExtension class, in Godot 4, you can attach scripts by selecting the class from the extension.

For GDExtension classes:
- In the scene, select the node
- In Inspector > Node > Script, click the dropdown and select "Load"
- Browse to the extension's .gdextension file or select the class from the extension

Then run the scene (F6 or Play button).

This will instantiate `ExampleSceneScala`, add it to the scene tree, call `_ready()`, and print "Hello, Scala!" to Godot's output.

## Validation
After following the steps above, run Godot with the scene and check the output for "Hello, Scala!". The extension loading logs will still appear, followed by the scene's `_ready()` output.
